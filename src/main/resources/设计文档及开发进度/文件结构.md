# RAG 知识助手 Demo 后端文件架构说明

本文档定义后端 Java 项目的目录结构与文件组织方式，遵循 Spring Boot 约定与分层架构原则。各功能模块以独立包的形式组织，包内类职责单一，便于理解、开发与后续扩展。

## 1. 整体目录结构

| 目录 | 作用 |
|:-----|:-----|
| `RAGApplication.java` | Spring Boot 应用启动入口，包含 `main` 方法，负责启动整个应用。 |
| `config/` | 配置模块，集中管理配置属性与 Bean 定义。 |
| `conversation/` | 会话管理模块，负责对话会话的存储与查询。 |
| `llm/` | LLM 接入模块，封装大语言模型调用入口。 |
| `knowledge/` | 知识库管理模块，负责知识条目的全生命周期管理。 |
| `rag/` | RAG 主流程模块，串联检索与生成的编排单元。 |
| `common/` | 通用组件模块，存放跨模块共享的工具类与统一响应结构。 |
| `exception/` | 异常定义与全局处理模块。 |

## 2. 各包职责与文件说明

### 2.1 启动入口（根包）

| 文件 | 作用 |
|:-----|:-----|
| `RAGApplication.java` | Spring Boot 应用主入口类，包含 `main` 方法，负责启动整个应用。 |

### 2.2 配置模块（`config/`）

该包集中管理所有配置属性与 Bean 定义，将 `application.yml` 中的参数映射为类型安全的 Java 对象，并初始化 Spring AI 相关组件。

| 文件 | 作用 |
|:-----|:-----|
| `AppProperties.java` | 应用自定义配置类，绑定 `app.rag` 前缀的属性，包含检索数量、相似度阈值、提示词模板、文件上传路径等字段。 |
| `LLMConfig.java` | LLM 相关配置类，负责创建 `ChatClient.Builder` Bean，配置 Ollama 连接参数与模型名称。 |
| `VectorStoreConfig.java` | 向量存储配置类，负责创建 `VectorStore` Bean，配置 ChromaDB 连接地址与集合名称。 |
| `ChatMemoryConfig.java` | 会话记忆配置类，配置 `JdbcChatMemoryRepository` Bean，启用基于 H2 的会话持久化。 |

### 2.3 会话管理模块（`conversation/`）

该包负责对话会话的存储与查询，复用 Spring AI 的会话记忆抽象，为前端提供会话列表与历史消息的 REST 接口。

| 文件 | 作用 |
|:-----|:-----|
| `ConversationController.java` | REST 控制器，暴露会话列表查询、历史消息获取、删除会话等 HTTP 接口。 |
| `ConversationService.java` | 业务服务类，封装对 `ChatMemory` 和 `ChatMemoryRepository` 的操作逻辑。 |
| `dto/ConversationDto.java` | 数据传输对象，定义返回给前端的会话信息结构。 |
| `dto/MessageDto.java` | 数据传输对象，定义返回给前端的单条消息结构。 |

### 2.4 LLM 接入模块（`llm/`）

该包对 Spring AI 的 `ChatClient` 进行薄封装，提供统一的流式生成调用入口，支持通过配置切换底层模型服务。

| 文件 | 作用 |
|:-----|:-----|
| `LLMClientService.java` | 封装 `ChatClient` 的流式调用逻辑，提供 `streamResponse` 方法供上层调用。 |
| `client/OllamaClientProperties.java` | （可选）Ollama 连接参数配置类，用于扩展自定义配置项。 |

### 2.5 知识库管理模块（`knowledge/`）

该包负责知识条目的全生命周期管理，包括文档解析、索引构建、手动录入、分页查询、编辑与删除。内部进一步划分为控制器层、服务层与数据访问抽象。

| 文件 | 作用 |
|:-----|:-----|
| **控制器层** | |
| `KnowledgeController.java` | REST 控制器，暴露文档上传、手动录入、条目分页查询、详情获取、更新、删除等 HTTP 接口。 |
| **服务层** | |
| `DocumentService.java` | 文档处理服务，集成 Apache Tika，负责从上传文件中提取纯文本内容。 |
| `IndexService.java` | 索引构建服务，负责文本分块、调用 Embedding 并写入 `VectorStore`。 |
| `KnowledgeItemService.java` | 知识条目管理服务，提供分页查询、详情获取、更新（含重新向量化）、删除等业务逻辑。 |
| **数据传输对象** | |
| `dto/UploadResponse.java` | 文档上传接口的响应结构，包含处理结果与条目标识。 |
| `dto/ManualEntryRequest.java` | 手动录入接口的请求体结构，包含标题、内容、标签等字段。 |
| `dto/KnowledgeItemDto.java` | 知识条目分页列表与详情接口的响应结构，包含摘要、来源、创建时间等信息。 |
| `dto/KnowledgeUpdateRequest.java` | 更新条目接口的请求体结构。 |

### 2.6 RAG 主流程模块（`rag/`）

该包是系统的核心编排单元，负责将向量检索与 LLM 生成串联为一个完整的问答流程。

| 文件 | 作用 |
|:-----|:-----|
| `RAGService.java` | 核心编排服务，在初始化时将 `VectorStore` 以 `QuestionAnswerAdvisor` 形式绑定至 `ChatClient`。对外提供 `query` 流式方法，接收用户问题并返回 SSE 流式响应。 |
| `advisor/CustomQuestionAnswerAdvisor.java` | （可选）自定义检索顾问，用于扩展默认的检索逻辑，例如添加元数据过滤或重排序。 |

### 2.7 通用组件模块（`common/`）

该包存放跨模块共享的通用代码，包括统一响应格式、工具类等。

| 文件 | 作用 |
|:-----|:-----|
| `dto/ApiResponse.java` | 统一 API 响应结构，包含状态码、消息、数据字段，用于所有非流式接口。 |
| `util/FileUtils.java` | 文件操作工具类，提供文件保存、删除等通用方法。 |
| `util/JsonUtils.java` | JSON 序列化与反序列化工具类。 |

### 2.8 异常处理模块（`exception/`）

该包定义业务异常类型与全局异常处理器，统一处理各类异常并返回符合 `ApiResponse` 结构的错误响应。

| 文件 | 作用 |
|:-----|:-----|
| `RAGException.java` | 自定义运行时异常基类，用于标识业务逻辑异常。 |
| `DocumentParseException.java` | 文档解析异常，当 Tika 无法提取文本时抛出。 |
| `IndexBuildException.java` | 索引构建异常，当向量化或写入 ChromaDB 失败时抛出。 |
| `GlobalExceptionHandler.java` | 全局异常处理器，使用 `@RestControllerAdvice` 拦截各类异常并返回统一错误响应。 |

## 3. 资源文件目录

| 路径 | 作用 |
|:-----|:-----|
| `src/main/resources/application.yml` | Spring Boot 主配置文件。 |
| `src/main/resources/application-dev.yml` | （可选）开发环境配置文件。 |
| `src/main/resources/prompts/rag-prompt.txt` | RAG 提示词模板文件，可被 `AppProperties` 引用。 |
| `src/main/resources/static/` | 前端静态资源目录，包含对话工作台与知识库管理页面的 HTML、CSS、JS 文件。 |
| `src/main/resources/static/index.html` | 对话工作台页面。 |
| `src/main/resources/static/knowledge.html` | 知识库管理中心页面。 |
| `src/main/resources/static/css/` | 样式文件存放目录。 |
| `src/main/resources/static/js/` | JavaScript 脚本存放目录。 |
| `src/main/resources/db/` | H2 数据库文件存放目录（自动生成）。 |

## 4. 包依赖关系

依赖方向遵循自上而下的单向依赖原则，上层可依赖下层，下层不感知上层。

- `config` 包被所有业务模块依赖，提供配置属性与核心 Bean。
- `llm` 包被 `rag` 包和 `knowledge` 包的索引服务依赖。
- `knowledge` 包和 `conversation` 包相互独立，无直接依赖。
- `rag` 包依赖 `knowledge` 包中的 `VectorStore` Bean，同时依赖 `llm` 包。
- `common` 包被所有包依赖。
- `exception` 包被所有包依赖。

## 5. 扩展指南

当需要扩展功能时，可在对应包内新增类，或按以下方式创建新包：

- 如需增加新的 LLM 提供商，在 `llm/client/` 下新增实现类，并调整 `LLMConfig` 的条件注入逻辑。
- 如需增加重排序功能，在 `rag/rerank/` 下新增 `Reranker` 接口与实现，并在 `RAGService` 中注入调用。
- 如需增加更复杂的分块策略，在 `knowledge/splitter/` 下新增 `TextSplitter` 接口与多种实现。

本文档为开发者提供了清晰的文件导航与职责说明，确保项目结构在开发迭代中保持整洁与可维护性。
